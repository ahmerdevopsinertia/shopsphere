Create ADR
docs/architecture/adr/

ADR-007-user-profile-separation.md

Decision:

User authentication information and personal profile information will be stored separately using a one-to-one relationship.

Update Prisma Schema

Add:

model UserProfile {
  id        String   @id @default(uuid())
  userId    String   @unique

  firstName String
  lastName  String

  phone     String?
  avatar    String?

  user User @relation(
    fields: [userId],
    references: [id]
  )

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

And update User:

model User {

  ...

  profile UserProfile?
}
